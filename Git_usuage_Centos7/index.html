关于git

  • git安装：

  • git代码管理，分为四层：

    • 第一层：目录中的文件
    • 第二层：缓存区，提交命令git add file后，相应文件暂存在缓存区。
    • 第三层：本地仓库，提交命令git commit -m “message”后，缓存区的最新版本文件就会存放在本地仓库。在git中版本更新通过HEAD的指向进行表示，每一次更新或回退可以更改HEAD的指向，但是仓库中的每一个版本都还存在。
    • 第四层：远程仓库，执行push之后将本地仓库上传到远程仓库github中。

  • 常用命令：

    • git init:初始化本地目录，上传前的第一步
    • git status:查看当前仓库的状态
    • git add . ：将本地目录下的文件提交到暂存区
    • git commit -m “message”：将本地缓存区提交到本地仓库
    • git log：查看当前版本之前的提交记录
    • git remote rm origin:删除当前分支，重复提交需要用到
    • git diff:查看当前缓存区与文件差别

git使用

  • git初始化，首次安装配置：

    • 配置电子邮件和用户名

    • 配置文本编辑器

    • 生成公钥，并在github中添加生成的公钥(不是私钥)

    • 将生成的id_rsa.pub文件中的内容添加到github上，文件可能不可见在终端在切换到.ssh目录下可见。

  • 提交到远程仓库：

    • 从远程仓库clone：git clone git@github.com:username/projectname.git
